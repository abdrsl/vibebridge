# Session管理功能总结

## 解决的问题

### 原始问题
飞书每次请求都会新开一个session，导致：
1. **上下文丢失**：每次对话都是独立的，无法记住之前的对话历史
2. **用户体验差**：用户需要重复说明需求
3. **资源浪费**：每次都要重新初始化OpenCode

### 解决方案
实现了完整的Session管理系统，支持：
1. **Session持久化**：为每个飞书对话维护session状态
2. **用户确认机制**：任务执行前需要用户确认
3. **会话连续性**：任务完成后可以继续同一session
4. **状态管理**：完整的session生命周期管理

## 实现的功能

### 1. Session管理器 (`app/session_manager.py`)
- **Session状态管理**：PENDING, CONFIRMED, RUNNING, COMPLETED, FAILED, CANCELLED, EXPIRED
- **消息历史记录**：存储用户和助手的对话历史
- **自动过期清理**：1小时自动过期，定期清理
- **持久化存储**：JSON文件存储，支持重启恢复

### 2. 交互式卡片系统 (`app/feishu_client.py` 扩展)
- **确认卡片**：新任务需要用户确认执行
- **继续卡片**：任务完成后询问是否继续
- **状态卡片**：显示session状态和操作按钮
- **操作卡片**：支持确认、取消、继续、新建等操作

### 3. Webhook处理器 (`app/feishu_webhook_handler.py`)
- **智能路由**：根据session状态决定处理逻辑
- **用户确认流程**：新任务 → 确认卡片 → 用户确认 → 执行
- **会话继续流程**：已完成任务 → 继续卡片 → 用户选择 → 继续/新建
- **状态管理**：支持状态查询和任务取消

### 4. 与OpenCode集成
- **Session关联**：OpenCode任务与session关联
- **状态同步**：任务状态自动同步到session
- **连续执行**：同一session可以执行多个相关任务

## 工作流程

### 新用户请求流程
```
用户发送消息 → 创建新session → 发送确认卡片 → 
    ↓
用户确认 → 开始OpenCode任务 → 更新session状态 →
    ↓
任务完成 → 更新session为COMPLETED → 等待用户新消息
```

### 继续会话流程
```
用户发送消息 → 检查已有session → 发送继续卡片 →
    ↓
用户选择"继续" → 使用同一session → 开始新任务 →
    ↓
用户选择"新建" → 关闭旧session → 创建新session
```

### 状态管理流程
```
用户发送"/status" → 列出所有session → 发送状态卡片 →
    ↓
用户发送"/cancel" → 取消运行中任务 → 更新session状态
```

## 文件发送功能的集成

Session管理系统与文件发送功能完美集成：

### 1. **文件发送作为任务**
- 创建HTML文件 → 发送到飞书 → 这是一个完整的OpenCode任务
- 任务状态自动同步到session
- 支持任务重试和继续

### 2. **用户确认机制**
- 用户请求"做一个漂亮的html个人网页发过来"
- 系统发送确认卡片询问是否执行
- 用户确认后开始创建和发送文件

### 3. **会话连续性**
- 文件发送完成后，session保持活跃
- 用户可以继续请求其他文件操作
- 所有操作在同一上下文中

## 配置和使用

### 环境变量
```bash
# 飞书配置
export FEISHU_APP_ID=your_app_id
export FEISHU_APP_SECRET=your_app_secret
export FEISHU_DEFAULT_CHAT_ID=your_chat_id

# Session配置（可选）
export SESSION_STORAGE_PATH=data/sessions  # 默认
export SESSION_EXPIRE_HOURS=1              # 默认
```

### API端点
- `POST /feishu/webhook/opencode` - 处理飞书消息（支持session）
- `GET /opencode/tasks` - 查看OpenCode任务
- `POST /opencode/tasks` - 创建OpenCode任务（直接API）

### 用户命令
- `帮助` / `help` - 显示帮助信息
- `状态` / `status` - 查看session状态
- `取消` / `cancel` - 取消当前任务
